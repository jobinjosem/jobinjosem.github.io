WSL BASH Customization

  2022-03-01 Word Count: 304 words Read Time: 2 minutes

Customize your Windows Terminal BASH in WSL2

There are thousands of articles available in the internet about how you can customize your bash prompt in different ways. In this post, I will explain how to personalize your widows terminal bash prompt in WSL2 from a Devops engineer viewpoint.

Often times we have to interact with Cloud resources in Azure and Kubernetes using different az cli and kubectl. How about showing the cluster you are connected, the namespace you are in, the current azure subscription and the git repository you are working in and it’s current branch all in one place, awesome right? Let me explain how I did this.

There are some prerequisites you have to install to get this working.

  1. Install kubectl + git
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
sudo apt-get install git
  1. Copy kube-ps1.sh from kube-ps1
git clone https://github.com/jonmosco/kube-ps1
mv kube-ps1/kube-ps1.sh ~
rm -rf kube-ps1
  1. Obtain kubectx + kubens from kubectx
sudo git clone https://github.com/ahmetb/kubectx /opt/kubectx
sudo mv /opt/kubectx/kubectx /usr/local/bin/kubectx
sudo mv /opt/kubectx/kubens /usr/local/bin/kubens
rm -rf /opt/kubectx
  1. Install fzf
git clone --depth 1 https://github.com/junegunn/fzf.git ~/.fzf
~/.fzf/install
  1. Install jq
sudo apt update
sudo apt install -y jq
  1. Edit your [~/.bashrc] file

Add the below lines to your .bashrc file.

source ~/kube-ps1.sh
alias n='kubens'
alias x='kubectx'
PS1='$(kube_ps1)'"\[\e[33m\] \D{%H:%M} \[\033[95m\][\$(az account show 2>/dev/null | jq -r .name)] \[\e[32m\]\W \[\033[33m\][\$(git symbolic-ref --short HEAD 2>/dev/null)] \[\e[00m\]$ "
  1. Relaunch your interactive shell instane
source ~/.bashrc

You can easily change your kubernetes cluster by typing x and pressing enter. A list of available Kubernetes clusters in your kubecfg will be shown.

Similarly pressing n will list all the namespaces in the selected cluster and you can select from the list.

You are done. The terminal must look like this!!
